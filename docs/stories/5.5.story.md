# Story 5.5 — Métricas Individuais do Cliente: CAC, LTV, Payback e Tenure

**Epic:** 9 — Visão Financeira Avançada do Cliente
**Status:** Done
**PRD refs:** FR-34, FR-63, FR-63b, FR-63c, FR-63d, FR-63e, FR-63f

## Descrição
Como gestor, quero ver no perfil do cliente um painel de métricas individuais — margem de contribuição, ARR, CAC, LTV/CAC ratio, payback period e tenure — para avaliar a saúde e rentabilidade de cada cliente sem cruzar dados manualmente.

## Acceptance Criteria

- [ ] Card "Métricas do Cliente" na tab Financeiro exibe:
  - **Margem de Contribuição**: Σ(valor_contratado − custo_base) dos produtos ativos (FR-63)
  - **ARR do cliente**: MRR do cliente × 12 (FR-63b)
  - **CAC**: campo `custo_aquisicao` do cliente, editável pelo admin inline (FR-63c)
  - **LTV**: soma histórica de `pagamentos.valor_pago` confirmados do cliente
  - **LTV/CAC ratio**: LTV ÷ CAC — semáforo visual: verde (≥ 5×), âmbar (3–5×), vermelho (< 3×) (FR-63d)
  - **Payback Period**: CAC ÷ MRR do cliente, resultado em meses (FR-63e)
  - **Tenure**: meses desde `data_inicio_relac` (ou `created_at` se null) até hoje (FR-63f)
  - **Ticket Médio Mensal**: MRR do cliente (soma `valor_efetivo` dos itens recorrentes ativos) — base para Payback Period (FR-34)
- [ ] Admin pode editar `custo_aquisicao` diretamente no card (campo inline com confirmação)
- [ ] Métricas com dados insuficientes exibem "—" em vez de erro (ex: CAC null, MRR zero)
- [ ] Semáforo LTV/CAC com tooltip explicando os benchmarks (< 3× ruim, 3–5× ok, > 5× ótimo)

## Tarefas Técnicas

- [ ] Server Action: `updateCustoAquisicao(cliente_id, valor)` — admin only
- [ ] Componente `metricas-cliente-card.tsx`: card com todos os KPIs calculados
- [ ] Query: MRR do cliente — soma `valor_efetivo` de `produtos_contratados` view onde `produto_tipo = 'recorrente'`
- [ ] Query: LTV do cliente — soma `valor_pago` de `pagamentos` onde `status = 'confirmado'`
- [ ] Query: Margem de Contribuição — JOIN `contrato_itens` + `produtos_agencia` para `valor_negociado − custo_base`
- [ ] Adicionar `metricas-cliente-card.tsx` na `financeiro-tab.tsx` (abaixo do banner de próxima fatura)

## Notas Técnicas
- Sem migration: `custo_aquisicao` e `data_inicio_relac` já existem em `clientes`
- Todos os KPIs são calculados em runtime — sem persistência adicional
- Payback: se MRR = 0, exibir "—" (cliente sem produtos recorrentes ativos)
- LTV/CAC: se CAC = null ou 0, exibir "—" (CAC não informado)
- Tenure: preferir `data_inicio_relac` sobre `created_at` — pode ser retroativo ao início real da relação

## Definição de Pronto
- Card exibe todos os 7 KPIs com valores reais ou "—" quando indisponível
- Admin consegue editar CAC inline
- Semáforo LTV/CAC com tooltip funciona corretamente

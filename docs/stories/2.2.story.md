# Story 2.2 — Variantes e Planos por Produto

**Epic:** 2 — Catálogo de Produtos e Serviços
**Status:** Done
**PRD refs:** FR-11

## Descrição
Como admin, quero definir variantes/planos para cada produto do catálogo (ex: Basic / Pro / Enterprise), para que as contratações possam referenciar um plano específico com valor, periodicidade e condições comerciais próprias.

## Acceptance Criteria

- [x] Admin pode criar variantes (ofertas) para um produto existente
- [x] Cada variante tem: nome, valor, periodicidade, carência (meses), prazo de aviso de cancelamento
- [x] Cada variante tem condições comerciais: multa (tipo + valor), índice de reajuste, renovação automática
- [x] Variantes são listadas dentro do produto na página `/produtos`
- [x] Admin pode ativar/desativar uma variante sem afetar contratações existentes

## Tarefas Técnicas

- [x] Server Action: `createOferta`, `updateOferta`, `toggleOfertaAtivo`
- [x] Expandir `produto-drawer.tsx`: seção de variantes com lista + botão adicionar
- [x] Componente `oferta-drawer.tsx` (drawer para criar/editar variante)

## Notas Técnicas
- Schema já existe: tabela `produto_ofertas` com todos os campos necessários — sem migration
- `oferta_id` já está em `contrato_itens` como nullable — contratação sem variante é válida
- Campos de multa e reajuste na oferta têm precedência sobre o contrato (quando preenchidos)
- `renovacao_automatica` e `dias_geracao_fatura` são configurados por oferta

## Definição de Pronto
- Admin consegue criar produto com 2 variantes (ex: Basic e Pro)
- Variantes listadas dentro do produto em `/produtos` com toggle ativo/inativo

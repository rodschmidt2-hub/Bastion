# Story 1.1 — Contatos Múltiplos por Cliente

**Epic:** 1 — Cadastro e Ciclo de Vida do Cliente
**Status:** Done
**PRD refs:** FR-02

## Descrição
Como gestor, quero registrar múltiplos contatos por cliente com cargo e flags de função, para saber quem acionar para cobrança, NF-e e decisões.

## Acceptance Criteria

- [x] Cliente pode ter N contatos cadastrados
- [x] Cada contato tem: nome, cargo (texto livre), whatsapp, email
- [x] Flags de função por contato: Principal, Cobrança, NF-e (múltiplos podem ser marcados)
- [x] Apenas um contato pode ser `is_principal` por cliente — marcar outro remove o anterior
- [x] Tab Dados do cliente exibe seção "Contatos" com lista e botão adicionar
- [x] Formulário inline para adicionar/editar contato (sem drawer separado)
- [x] Exclusão de contato com confirmação
- [x] Contato principal aparece em destaque no card do cliente na listagem

## Tarefas Técnicas

- [x] Server Action: `createContato`, `updateContato`, `deleteContato`, `setPrincipal`
- [x] Componente `contatos-section.tsx` dentro de `dados-tab.tsx`
- [x] Atualizar `clientes-table.tsx` para mostrar nome + whatsapp do contato principal

## Notas Técnicas
- Schema já existe: `contatos_cliente` com `nome`, `cargo`, `email`, `whatsapp`, `is_principal`, `is_cobranca`, `is_nfe`
- Sem migration necessária
- `setPrincipal` deve fazer UPDATE em todos os contatos do cliente (is_principal = false) + SET no escolhido — usar transação
- `is_cobranca` e `is_nfe` serão usados pela Story 5.1 para roteamento de faturas

## Definição de Pronto
- CRUD de contatos funcional na tab Dados
- Contato principal visível na listagem de clientes
- Flags de cobrança e NF-e persistem corretamente

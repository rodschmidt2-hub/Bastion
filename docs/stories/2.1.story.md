# Story 2.1 — Expandir Catálogo de Produtos

**Epic:** 2 — Catálogo de Produtos e Serviços
**Status:** Done
**PRD refs:** FR-09, FR-10, FR-13, FR-14, FR-14b

## Descrição
Como admin, quero que o catálogo de produtos tenha tipo (recorrente/pontual/híbrido), periodicidade e custo base, para que as contratações gerem faturas corretamente e eu possa calcular margem de contribuição.

## Acceptance Criteria

- [x] Campo `tipo` no produto: `recorrente` | `pontual` | `híbrido`
- [x] Campo `periodicidade` no produto: `mensal` | `trimestral` | `semestral` | `anual` (obrigatório se tipo = recorrente ou híbrido)
- [x] Campo `custo_base` no produto: valor decimal opcional (admin-only) — usado para cálculo de margem
- [x] Página `/produtos` acessível apenas para admin
- [x] Listagem de todos os produtos do catálogo com status ativo/inativo
- [x] Formulário para criar novo produto (nome, categoria, tipo, periodicidade, valor base, custo base, descrição)
- [x] Formulário para editar produto existente
- [x] Toggle ativar/desativar produto — não afeta contratações existentes
- [x] Produtos inativos ficam visíveis na lista com badge "Inativo"

## Tarefas Técnicas

- [x] Server Action: `createProduto`, `updateProduto`, `toggleProdutoAtivo`
- [x] Página `/produtos/page.tsx` com tabela + botão novo produto
- [x] Componente `produto-drawer.tsx` (drawer lateral para criar/editar)
- [x] Adicionar "Produtos" no sidebar (apenas admin)
- [x] Criar seed inicial com produtos de exemplo (recorrente mensal, pontual, híbrido)

## Notas Técnicas
- Schema já existe: tabela `produtos_agencia` com `tipo`, `periodicidade`, `custo_base` — sem migration necessária
- `tipo` aceita `híbrido` além de recorrente/pontual (definido no schema)
- `periodicidade` só aparece no formulário quando tipo = recorrente ou híbrido
- `custo_base` é campo interno — não exibido para o cliente
- Produtos inativos não devem aparecer no dropdown de "adicionar produto ao cliente"

## Definição de Pronto
- Página carrega sem erro
- CRUD completo funcional
- Admin consegue criar produto recorrente mensal com custo_base e vê na lista

# Story 6.4 — Projeção de Receita (Próximos 3 Meses)

**Epic:** 6 — Dashboard e Métricas
**Status:** Done
**PRD refs:** FR-44

## Descrição
Como admin, quero ver uma projeção de receita para os próximos 3 meses baseada nos produtos recorrentes ativos e renovações previstas, para planejar o caixa da agência.

## Acceptance Criteria

- [ ] Card "Projeção de Receita" no dashboard com 3 colunas: próximo mês, mês +2, mês +3
- [ ] Cada coluna exibe: receita projetada total + quantidade de clientes ativos projetados
- [ ] Projeção considera: produtos recorrentes sem `data_fim_item` (open-end) como garantidos
- [ ] Produtos com `data_fim_item` no período projetado são marcados como "em risco" e excluídos da projeção base
- [ ] Seção "Receita em Risco": lista de contratos que vencem nos próximos 3 meses com valor e cliente
- [ ] Projeção não inclui produtos pontuais (one-time)

## Tarefas Técnicas

- [ ] Query: projeção por mês — `produtos_contratados` view WHERE `produto_tipo = 'recorrente'` AND (`data_fim_item IS NULL` OR `data_fim_item > {fim_do_mes_alvo}`)
- [ ] Query: receita em risco — itens com `data_fim_item` entre hoje e +90 dias, GROUP BY mês
- [ ] Componente `projecao-receita-card.tsx`: 3 colunas de projeção + lista de risco
- [ ] Adicionar card no `dashboard/page.tsx` na seção de métricas

## Notas Técnicas
- Sem migration
- Projeção é conservadora: inclui apenas itens garantidos (sem data_fim ou data_fim > período)
- Não considera churn futuro, expansão ou novos contratos — é baseline
- "Receita em risco" = valor que sai se o contrato não for renovado
- Cálculo feito em runtime — sem persistência

## Definição de Pronto
- Card exibe projeção dos 3 próximos meses com valores reais
- Lista de receita em risco mostra contratos a vencer
- Projeção exclui corretamente produtos pontuais e itens vencidos

# Story 10.1 — NPS/CSAT por Cliente

**Epic:** 10 — NPS e Satisfação do Cliente
**Status:** Done
**PRD refs:** FR-64, FR-65, FR-66

## Descrição
Como gestor, quero registrar manualmente a nota NPS de cada cliente com data e observação, e ver o histórico de evolução ao longo do tempo, para monitorar a satisfação da base.

## Acceptance Criteria

- [x] Gestor pode registrar nota NPS: score 0–10 + data + observação (texto livre)
- [x] Múltiplos registros por cliente ao longo do tempo — cada entrada é histórico, não sobrescreve
- [x] Tab "NPS" no perfil do cliente exibe histórico em ordem cronológica reversa
- [x] Cada entrada exibe: score, data, responsável que registrou, observação
- [x] Score atual em destaque no topo da tab: última nota + variação em relação à anterior (↑↓)
- [x] Badge NPS no card do cliente na listagem: verde (≥ 9), âmbar (7–8), vermelho (≤ 6), cinza (sem registro)
- [x] Exclusão de registro com confirmação (apenas admin)

## Tarefas Técnicas

- [x] Server Action: `createNpsRegistro(cliente_id, score, data_registro, observacao)`
- [x] Server Action: `deleteNpsRegistro(id)` — admin only
- [x] Componente `nps-tab.tsx`: score atual em destaque + lista histórico
- [x] Atualizar `clientes-table.tsx`: badge NPS com cor por faixa de score
- [x] Query: última nota por cliente para badge na listagem (usar `DISTINCT ON` ou subquery)

## Notas Técnicas
- Schema já existe: `nps_registros` com `score` (0–10), `data_registro`, `responsavel_id`, `observacao`
- Sem migration necessária
- Variação: score_atual − score_anterior (null se primeiro registro)
- Faixas NPS: Promotores ≥ 9, Neutros 7–8, Detratores ≤ 6

## Definição de Pronto
- Gestor registra NPS e vê histórico na tab
- Badge com cor aparece na listagem de clientes
- Score atual e variação visíveis no topo da tab

# Story 3.1 — Produtos Contratados com Ciclo de Renovação

**Epic:** 3 — Produtos Contratados (Subscriptions)
**Status:** Done
**PRD refs:** FR-18, FR-19, FR-20, FR-22

## Descrição
Como gestor, quero visualizar os produtos contratados de cada cliente com sua data de vencimento e conseguir renovar manualmente, para ter controle do ciclo de vida de cada item contratado.

## Acceptance Criteria

- [x] Tab "Produtos" no perfil do cliente lista itens via view `produtos_contratados`
- [x] Cada item exibe: produto, valor efetivo, periodicidade, status do contrato, data de vencimento (`data_fim_item`)
- [x] Badge de alerta na coluna "Vencimento": vermelho se <= 7 dias, âmbar se <= 30 dias, sem badge se > 30 dias
- [x] Produtos com `data_fim_item = NULL` (open-end / recorrente sem fim definido) exibem "Sem vencimento"
- [x] Produto com `produto_tipo = pontual` não exibe badge de vencimento nem botão Renovar
- [x] Botão "Renovar" em cada item recorrente: abre modal com nova data + campo de observações
- [x] Renovação atualiza `data_fim_item` no `contrato_item` correspondente
- [x] Botão "Adicionar Produto" na tab Produtos: seleciona produto do catálogo, define valor negociado, data início e oferta (opcional) → cria `contrato_item` vinculado ao contrato ativo do cliente
- [x] Seletor de oferta no formulário de adição: lista variantes ativas do produto selecionado (opcional — produto pode ser adicionado sem variante)
- [x] Variantes inativas não aparecem no seletor de oferta
- [x] Status "em_cancelamento" disponível — gestor pode marcar item para cancelamento futuro
- [x] Página global `/contratos` exibe alertas de itens a vencer nos próximos 30 dias (todos os clientes)

## Tarefas Técnicas

- [x] Server Action: `adicionarProdutoCliente(contrato_id, produto_id, oferta_id, valor_negociado, data_inicio)` → insere em `contrato_itens`
- [x] Server Action: `renovarContratoItem(id, nova_data_fim, observacoes)`
- [x] Server Action: `atualizarStatusContratoItem(id, status)`
- [x] Componente `produtos-tab.tsx`: tabela com colunas produto / valor / periodicidade / vencimento / status / ações + botão "Adicionar Produto"
- [x] Componente `adicionar-produto-modal.tsx`: seletor de produto, oferta, valor e data
- [x] Componente `renovacao-modal.tsx`: date picker + textarea observações + confirmação
- [x] Atualizar `/contratos/page.tsx`: seção de alertas de vencimento próximo (query sobre `produtos_contratados` view)

## Notas Técnicas
- Schema: `contrato_itens` é a tabela real; `produtos_contratados` é view que já faz os joins necessários
- `periodicidade` vem de `produtos_agencia` via view — não está em `contrato_itens`
- `valor_efetivo` já calculado na view (considera descontos)
- Renovação salva observações no `audit_log` (via trigger existente) — histórico completo fica na Story 3.2
- Sem migration necessária — schema completo já existe

## Definição de Pronto
- Tab Produtos mostra itens com vencimento e badges corretos
- Admin consegue adicionar produto ao cliente com oferta opcional
- Renovação atualiza `data_fim_item` e confirma ao usuário
- `/contratos` exibe alertas de vencimento próximo

# Story 5.3 — MRR Histórico Mês a Mês

**Epic:** 5 — Financeiro e Cobranças
**Status:** Done
**PRD refs:** FR-35

## Descrição
Como admin, quero ver a evolução do MRR mês a mês nos últimos 12 meses, para acompanhar o crescimento da receita recorrente ao longo do tempo.

## Acceptance Criteria

- [ ] Gráfico de linha no dashboard e em `/financeiro` exibe MRR mensal dos últimos 12 meses
- [ ] Cada ponto do gráfico mostra: competência (Mês/Ano), MRR total, clientes ativos, margem %
- [ ] Job mensal captura snapshot do MRR no fechamento de cada mês
- [ ] Admin pode disparar snapshot manual via botão "Atualizar snapshot" (admin only)
- [ ] Meses sem snapshot exibem valor zero no gráfico (não quebram o eixo)

## Tarefas Técnicas

- [ ] Migration SQL: criar tabela `mrr_historico`:
  ```sql
  CREATE TABLE mrr_historico (
    id               UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    agencia_id       UUID         NOT NULL REFERENCES agencias(id),
    competencia      TEXT         NOT NULL,  -- YYYY-MM
    mrr              NUMERIC(12,2) NOT NULL,
    arr              NUMERIC(12,2) NOT NULL,
    clientes_ativos  INTEGER      NOT NULL,
    custo_total      NUMERIC(12,2),
    margem_percentual NUMERIC(5,2),
    created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    UNIQUE (agencia_id, competencia)
  );
  ALTER TABLE mrr_historico ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "tenant_isolation" ON mrr_historico USING (agencia_id = fn_agencia_id());
  ```
- [ ] Server Action: `snapshotMrr(agencia_id)` — lê `mrr_snapshot` e insere em `mrr_historico` (upsert por competência)
- [ ] Vercel Cron Job: `GET /api/cron/mrr-snapshot` — roda dia 1 de cada mês, chama `snapshotMrr`
- [ ] Componente `mrr-historico-chart.tsx` — gráfico de linha (Recharts) com últimos 12 meses
- [ ] Adicionar `mrr-historico-chart.tsx` na página `/financeiro` e no dashboard (Story 6.1 reutiliza)

## Notas Técnicas
- `mrr_snapshot` materialized view já existe — `snapshotMrr` lê dela e persiste em `mrr_historico`
- Cron Vercel: configurar em `vercel.json` com schedule `0 6 1 * *` (dia 1, 6h UTC)
- Backfill: primeiro deploy pode rodar `snapshotMrr` manualmente para popular meses anteriores via faturas
- Recharts já instalado na Story 5.2 — reutilizar

# Story 5.2 — Visão Financeira Avançada do Cliente

**Epic:** 9 — Visão Financeira Avançada do Cliente
**Status:** Done

**PRD refs:** FR-55, FR-56, FR-57, FR-58, FR-59, FR-60, FR-61, FR-62

## Descrição
Como gestor, quero uma tab financeira completa no perfil do cliente com próxima fatura, atrasos, pontualidade histórica, MRR individual e resumo anual — sem precisar cruzar dados manualmente.

## Acceptance Criteria

- [ ] Banner "Próxima Fatura" no topo: valor total, data de vencimento, breakdown por item (`fatura_itens`) (FR-57)
- [ ] Indicador de inadimplência em destaque: dias em atraso + saldo devedor total em aberto (FR-56)
- [ ] Timeline unificada: faturas + pagamentos + renovações em ordem cronológica reversa com filtros por tipo (FR-55)
- [ ] Gráfico de barras MRR mensal do cliente (últimos 7 meses) — soma `valor_efetivo` dos itens ativos (FR-59)
- [ ] Grid de pontualidade por mês: verde (pago até vencimento), âmbar (1–5 dias após), vermelho (> 5 dias ou pendente) + score % (FR-60)
- [ ] Resumo anual: total pago no ano atual vs. ano anterior com variação % (FR-58)
- [ ] Tabela de reajustes: renovações onde `valor_novo != valor_anterior` — data, de → para, variação % (FR-61)
- [ ] Campo de notas financeiras internas: textarea com auto-save, não visível ao cliente (FR-62)

## Notas Técnicas
- Única migration desta story: coluna `nota_financeira TEXT` em `clientes`
- Timeline: UNION de `faturas` (tipo='fatura') + `pagamentos` (tipo='pagamento') + `renovacoes` (tipo='renovacao') ordenados por `created_at DESC`
- Pontualidade: compara `pagamentos.data_pagamento` vs `faturas.data_vencimento` — join via `fatura_id`
- Reajustes: query em `renovacoes` (Story 3.2) com `valor_novo != valor_anterior`
- MRR mensal do cliente: query runtime sobre `contrato_itens` ativos do cliente agrupados por mês — independente de `mrr_historico` (que é global por agência, não por cliente)
- Nota financeira: debounce 800ms, sem botão salvar explícito
- Recharts: instalar uma vez — reutilizado na Story 6.1
- Métricas avançadas (Margem, ARR, CAC, LTV/CAC, Payback por cliente) → Story 5.5

## Definição de Pronto
- Tab Financeiro exibe todos os 8 componentes com dados reais
- Timeline mostra mix de faturas + pagamentos + renovações
- Gráfico MRR renderiza corretamente
- Grid de pontualidade calcula score baseado em faturas reais
- Nota financeira salva e persiste entre sessões
